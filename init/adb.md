# 📱 ADB over Wi-Fi — Повний гайд з налаштування

Покроковий гайд для підключення Android-пристрою через ADB по мережі Wi-Fi без USB-кабелю.

---

## 📋 Вимоги

- Встановлений [Android SDK Platform Tools](https://developer.android.com/studio/releases/platform-tools) (містить `adb`)
- Android-пристрій та ПК підключені до **однієї Wi-Fi мережі**
- На пристрої увімкнено **Режим розробника** та **USB-відлагодження**

### Як увімкнути Режим розробника

**Налаштування → Про телефон → Номер збірки** — натисни **7 разів** поспіль.  
Після цього з'явиться розділ **«Параметри розробника»** у меню Налаштувань.

---

## ⚙️ Частина 1 — Увімкнення ADB over Wi-Fi на пристрої

Залежно від версії Android та прошивки існує **два способи** увімкнення бездротового відлагодження.

---

### 🟢 Спосіб A — Нативна підтримка (Android 11+ / MIUI / деякі Android 10)

Деякі виробники та Android 11+ мають вбудовану опцію безпосередньо у Параметрах розробника.

#### Android 11 і вище — Wireless Debugging

1. Відкрий **Налаштування → Параметри розробника**
2. Знайди та увімкни **«Бездротове відлагодження» (Wireless Debugging)**
3. Підтвердь запит на дозвіл у діалоговому вікні

> Докладніше про підключення через цей метод — див. **Метод 2** нижче.

#### MIUI (Xiaomi) — вбудована опція

1. **Налаштування → Параметри розробника → Відлагодження по USB** — увімкни
2. Там само знайди **«ADB через Wi-Fi»** або **«Wireless ADB»** — увімкни
3. На екрані одразу відобразиться **IP:порт** для підключення

#### Samsung (One UI 5+) / Motorola / Nokia

Деякі прошивки Android 10–11 також мають вбудовану опцію:

1. **Налаштування → Параметри розробника**
2. Шукай: **«ADB через мережу»**, **«ADB by network»**, або **«Wireless debugging»**
3. Якщо опція є — увімкни її; пристрій покаже IP та порт

---

### 🔴 Спосіб B — Без нативної підтримки (Android 5–10, стокові прошивки)

Якщо у Параметрах розробника **немає** вбудованої опції бездротового ADB — використовуй сторонній APK.

#### Варіант B1 — ADB Wi-Fi (найпростіший)

**Додаток:** [ADB Wi-Fi](https://play.google.com/store/apps/details?id=com.ttxapps.wifiadb) від ttxapps

> ⚠️ Потребує **root-прав** на пристрої.

1. Встанови додаток з Google Play
2. Відкрий додаток — він автоматично запустить ADB у TCP/IP-режимі на порту **5555**
3. На екрані відобразиться IP-адреса пристрою
4. На ПК підключись:

```bash
adb connect <IP_ПРИСТРОЮ>:5555
```

---

#### Варіант B2 — ADB over Wi-Fi (без root, через USB один раз)

**Додаток:** [ADB over WiFi](https://play.google.com/store/apps/details?id=com.surendramaran.adboverwifi)  
або  
**Додаток:** [Wireless ADB Switch](https://play.google.com/store/apps/details?id=com.github.sryze.wireadbswitch)

> ✅ Не потребує root. Потрібне **одноразове USB-підключення** для першого запуску.

1. Встанови додаток
2. Підключи пристрій до ПК через **USB**
3. Відкрий додаток і натисни **«Enable»** / **«Увімкнути»**
4. Додаток виконає `adb tcpip 5555` автоматично
5. Відображений IP використай для підключення на ПК:

```bash
adb connect <IP_ПРИСТРОЮ>:5555
```

6. Від'єднай USB — з'єднання залишається активним

---

#### Варіант B3 — Ручний запуск через USB (без стороннього APK)

Якщо не хочеш встановлювати додатки — можна зробити все вручну через термінал:

1. Підключи пристрій через **USB**
2. Відкрий термінал на ПК та виконай:

```bash
adb tcpip 5555
```

3. Дізнайся IP-адресу пристрою:

```bash
adb shell ip addr show wlan0 | grep "inet "
```

4. Підключись по Wi-Fi:

```bash
adb connect <IP_ПРИСТРОЮ>:5555
```

5. Від'єднай USB і перевір:

```bash
adb devices
```

> ⚠️ **Увага:** після перезавантаження пристрою потрібно повторювати кроки 1–4, оскільки TCP/IP-режим скидається.

## 🔌 Частина 2 — Підключення з ПК

### Метод 1 — Класичний (Android < 11, після налаштування через USB або APK)

```bash
# Підключитись
adb connect 192.168.1.105:5555

# Перевірити список пристроїв
adb devices
```

Пристрій має відображатись як `192.168.1.105:5555 device`.

---

### Метод 2 — Wireless Debugging (Android 11+)

#### Крок 1 — Спарювання (одноразово)

У меню **Бездротове відлагодження** натисни **«Підключити за допомогою коду підключення»**.

На екрані з'являться:
- **IP та порт для паювання** (наприклад, `192.168.1.105:37493`)
- **6-значний PIN-код**

У терміналі на ПК:

```bash
adb pair 192.168.1.105:37493
```

Введи PIN-код, коли з'явиться запит:

```
Enter pairing code: 123456
Successfully paired to 192.168.1.105:37493
```

#### Крок 2 — Підключення

Після паювання на головному екрані **Бездротового відлагодження** побачиш **основний IP:порт** (інший від порту паювання).

```bash
adb connect 192.168.1.105:5555
```

#### Крок 3 — Перевірка

```bash
adb devices
```

## ✅ Корисні команди

| Команда | Опис |
|---|---|
| `adb devices` | Список підключених пристроїв |
| `adb shell` | Відкрити shell на пристрої |
| `adb install app.apk` | Встановити APK |
| `adb logcat` | Переглянути логи |
| `adb disconnect` | Від'єднати всі пристрої |
| `adb disconnect <IP>:5555` | Від'єднати конкретний пристрій |
| `adb kill-server` | Зупинити ADB сервер |
| `adb start-server` | Запустити ADB сервер |
| `adb tcpip 5555` | Перевести ADB у TCP/IP-режим |
| `adb shell ip addr show wlan0` | Дізнатись IP пристрою |

## ⚠️ Усунення проблем

### Пристрій не відображається після `adb connect`

```bash
adb kill-server
adb start-server
adb connect <IP>:5555
```

### Помилка `Connection refused`

- Переконайся, що обидва пристрої в **одній Wi-Fi мережі**
- Для Android < 11: повтори `adb tcpip 5555` з підключеним USB
- Перевір, чи не блокує з'єднання **файрвол** на ПК (порт TCP 5555)
- Спробуй вимкнути/увімкнути Wi-Fi на пристрої

### Пристрій показується як `offline`

```bash
adb disconnect <IP>:5555
adb connect <IP>:5555
```

### Після перезавантаження пристрою з'єднання зникає

- **Android 11+ (Wireless Debugging):** просто повтори `adb connect <IP>:<порт>` — паювання зберігається
- **Android < 11:** потрібно знову підключити USB та виконати `adb tcpip 5555`; для автоматизації використовуй **APK-рішення** з Варіанту B1/B2

### IP-адреса пристрою змінилась

Призначи **статичний IP** у налаштуваннях роутера (резервування за MAC-адресою) або у Wi-Fi налаштуваннях самого пристрою (тип IP → Статичний).

---

## 📌 Поради

- **Статичний IP** — щоб IP не змінювався після перезавантаження, задай його у налаштуваннях Wi-Fi на пристрої або зарезервуй у роутері.
- **Безпека** — ADB over Wi-Fi не шифрує трафік; використовуй лише у довіреній мережі; після роботи вимикай бездротове відлагодження.
- **Порт за замовчуванням** — `5555`; при підключенні кількох пристроїв можна використовувати різні порти (`5556`, `5557` тощо).

## 🗺️ Коротка таблиця вибору методу

| Версія Android | Root? | Рекомендований спосіб |
|---|---|---|
| Android 11+ | Не потрібен | Wireless Debugging (нативно) |
| Android 10 (деякі прошивки) | Не потрібен | Вбудована опція або USB → `adb tcpip` |
| MIUI / One UI з опцією | Не потрібен | Вбудована опція у Параметрах розробника |
| Android 5–10, стокова прошивка | Є root | ADB Wi-Fi (Варіант B1) |
| Android 5–10, стокова прошивка | Немає root | USB одноразово → APK (Варіант B2) або вручну (B3) |
